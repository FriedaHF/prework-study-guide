HTML

  • The head element contains information (metadata) that is not visible and helps to render the page correctly (includes elements such as title, style, base, link, meta, and script.
  • Metadata contains important information about accessiblity, search engines, and performance.
  • An attribute, which defines more information about an element, is always defined in the beginning element, never the closing element.
  • Charset sets character encoding.
  • Http-equiv sets an HTTP header.
  • Viewport sets the default view for the page.
  • !DOCTYPE html is a declaration that is not visible on the page, tells the browser what document to expect, and is not case sensitive alike all other declarations.
  • Child Elements are elements nested inside of other elements.
  • Parent Elements are elements that hold all the child elements.
  • Semantic HTML elements play an important role in making sure the webpage is accessable to a wide variety of users; the name describes the role in which they play.
  • Section elements are semantic elements that define a section in a document that contains thematic content.
  • div creates a division within a page, while span creates an inline division.
  • Visible HTML elements require an opening and closing tag.
  • br elements creates a line of empty space (a line break).
  • a header element appears at the top of the page and usually contains a navigation menu and/or introductory content.
  • a footer element is for text at the bottom of the page, usually containing the author, copyright, contact, sitemap, and navivgation.
  • alt attribute contains a text string that describes the appearance and functionality of the image.
  • src attribute defines the location, or source, of the image file.
  • link element creates the connection to the external file.
  • rel attribute specifies the relationship between the current document and the linked document resource.
  • href attribute specifies the location (URL) of the external resource.

CSS

  • Inline CSS apply to a specific HTML tag and styles an element directly in the HTML file by using the style attribute.
  • Internal style sheet contains CSS rules for the entire webpage located in the Head section.
  • The CSS Rule: contains a selector and a Declaration block - within the Declaration Block are Declarations - the Declaration inclues a property and it's value such as color: (property) and blue; (value).
  • margin property indicates how much space outside of an element.
  • padding adds spaces around the content inside of an element.
  • CSS box model includes the content, padding, border, and margin for each element.
  • class attribute allows to share a CSS rule to any element by assigning the rule to a class attribute with a class selector (which is another word).
  • .card is a class selector.

Git

  • git is a git statement; it is the first word used for any commands related to git
  • checkout moves the working branch to a new branch (switches branches)
  • cd navigates to a new directory or file
  • cd .. navigates backwards until starting position
  • pwd shows which depository or file currently in
  • code . opens the Visual Studio Code in that depository or file
  • -b creates a new branch
  • feature/ indicates that the branch is a feature
  • status checks which branch you are in and any changes that needs to be committed
  • add adds mods
  • -A adds all changes
  • pull receives a branch's mods into the local branch
  • push creates a remote branch or adds to an existing branch
  • origin is the source in the GitHub Repo
  • main is the master branch name

JavaScript

  • var is a keyword used to declare a variable;
  • the name of the variable (or container) = (which is the operator to assign specific data needed to be stored in the variable) the data it stores such as: var shape = "circle";
  • topics is a variable that contains a string that includes the topics studied; the strings are surrounded by "quotation marks";
  • to access the string, just refer back to the variable's name;
  • console.log method is used to output a message to the web console by adding an argument;
  • in order to see the console.log in action, Chrome DevTools needs to be used (by pressing the right click, going to inspect, the pressing the console tab);
  • the topics need to be defined AFTER using console.log(topics); to log it in the console - otherwise it would read undefined in the console;
  • conditional statements and loops can be used to control the order in which (or if) the code is executed;
  • if statements are used to introduce decision-making into the code; the computer will check if it is truthy - if it is trule or truthy then it will do whatever action between the {curly brackets} - if it is null/0 or falsy then it will ignore it and move on;
  • an array is a single cariable that is used to hold a group of (usually related) data; var shapes = ["triangle", "square", "pentagon", "circle"];
  • a for loop contains 3 parts: the first statement (the starting point) - var x = 0;, the second statement (condition) - x < shapes.length;, and the final statment (allows the array to iterate over each item) x++ -- so now the loop looks like: for(var x = 0; x < shapes.length; x++) { // code block }
  • Commenting out code makes it inactive without deleting it: put /* before and */ after the block of code;
  • a function is a set of instructions that tells the computer how to perform a certain task;
  • Use the function keyword then add the name of the function followed by parenthesis such as: function helloWorld();
  • Control flow is athe order in which a computer executes code in a script;
  • each line of JavaScript code ends with a semicolon;
